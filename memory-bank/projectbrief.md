# belief-field — project brief

## Purpose

A published belief-field artifact mapping the documented beliefs of **Nathan McCauley** (Co-Founder & CEO, Anchorage Digital), produced by the `/belief-field` skill. It turns a deep, source-cited corpus of his public statements into two linked, self-contained web views: an interactive **3D belief map** and a searchable, sortable **belief database / explorer**. Intended to be opened directly (or hosted statically) to explore what he believes, how strongly, how often he says it, and from which sources — with every quote traceable to its origin.

## Type

published-deploy — static, self-contained HTML/JS artifact under git, GitHub remote `Winnsolutionsadmin/belief-field`.

## Status

Published / active artifact. Canonical view is the 3D belief field on the full 32/33-belief, public-sanitized dataset; the older 2D atlas was dropped (commit `3c50bfc`).

## Key tech / structure

- **`index.html` + `belief-map.html`** — the 3D belief field. Three.js loaded from the unpkg CDN (needs internet to view). Inline engine is generated by `source/build.mjs` (concatenates `engine/layout|themes|ui-person|engine.js`) — do not hand-edit the inline engine.
- **`belief-explorer.html`** — the tabulated belief database (filter / search / sort UI; fonts from Google Fonts).
- **`data/belief-db.js`** — `window.BELIEF_DB`: the scored corpus (32 beliefs, 7 sectors, ~883 quotes, ~69 sources; resonance + safety axes; register legend A–E). PUBLIC-SAFE build — private 1:1 material excluded (generated by `build-public-db.py`).
- **`data/beliefs-map-db.js`** — `window.SUBJECT/SECTORS/CLUSTERS/BELIEFS/LINKS` for the 3D map (generated by `source/build-map-from-db.py`; do not hand-edit).
- **`data/source-links.js`** — `window.SOURCE_LINKS`: slug → source URL for provenance.

## Key constraints

- This repo holds the **generated public bundle only**; the `source/` build scripts that produce the HTML and map DB are not vendored here. Regenerate via the build pipeline rather than hand-editing artifacts.
- Public-sanitized: do not reintroduce private / 1:1 material into this published repo.
